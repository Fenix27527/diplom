# 📋 Personnel Records

**Автоматизированная система учёта персонала для малого и среднего бизнеса**

> 💡 Простое, быстрое и надёжное решение для ведения кадрового учёта без сложных настроек и ежемесячных платежей.

---

## 🚀 Возможности программы

✅ **Учёт сотрудников**  
- Добавление, редактирование и удаление записей о работниках  
- Хранение ФИО, даты рождения, подразделения, должности, ИНН, адреса, образования и других данных  

✅ **Управление подразделениями и должностями**  
- Ведение справочника штатных единиц  
- Указание численности, рабочих часов и тарифных ставок  

✅ **Работа с вакансиями и резюме**  
- Публикация открытых вакансий  
- Приём и хранение резюме кандидатов  
- Удобный выбор вакансии из выпадающего списка  

✅ **Поиск и фильтрация**  
- Мгновенный поиск по любому полю (ФИО, ИНН, должность и др.)  
- Фильтрация данных в реальном времени  

✅ **История изменений**  
- Автоматическое сохранение всех правок: кто, когда и что изменил  
- Просмотр старых и новых значений для любой записи  

✅ **Экспорт в Excel**  
- Выгрузка любой таблицы в формат `.xlsx`  
- Корректное отображение дат, чисел и длинных текстов  
- Автоподбор ширины колонок и форматирование заголовков  

✅ **Безопасность**  
- Авторизация пользователей (логин/пароль)  
- Локальное хранение данных — информация не уходит в интернет  
- Соответствие требованиям 152-ФЗ о персональных данных

## ⚙️ Установка

### Шаг 1. Установите SQL Server Express
1. Скачайте бесплатную редакцию: [SQL Server Express](https://www.microsoft.com/ru-ru/sql-server/sql-server-downloads)
2. Запустите установщик, выберите **«Базовая»** установка
3. Запомните имя экземпляра (обычно `.\SQLEXPRESS`)

### Шаг 2. Создайте базу данных
1. Откройте **SQL Server Management Studio (SSMS)**
2. Подключитесь к серверу (имя экземпляра из шага 1)
3. Откройте файл `DatabaseScript.sql` (идёт в комплекте)
4. Нажмите **«Выполнить»** (или `F5`)
5. Дождитесь сообщения «Команды успешно выполнены»

### Шаг 3. Запустите приложение
1. Откройте папку с программой
2. Запустите файл `PersonellRecords.exe`
3. При первом запуске введите данные администратора:
   - **Логин:** `admin`
   - **Пароль:** `admin`

> 🔐 Смените пароль администратора после первого входа!

## 🎯 Как пользоваться

### 🔐 Вход в систему
1. Введите логин и пароль
2. Нажмите **«Войти»**
3. Если данных нет — нажмите **«Регистрация»** для создания нового пользователя

### 👥 Работа с сотрудниками
1. В главном меню выберите **«Сотрудники»**
2. Для добавления: кликните по пустой строке в таблице и введите данные
3. Для редактирования: измените значение в ячейке
4. Для удаления: выделите строку(и) → кнопка **«Удалить»**
5. Нажмите **«Обновить»**, чтобы сохранить изменения в базе

### 🏢 Подразделения и вакансии
Работа аналогична разделу «Сотрудники».  
⚠️ При удалении подразделения убедитесь, что к нему не привязаны сотрудники.

### 📄 Экспорт в Excel
1. Откройте нужную таблицу (Сотрудники, Подразделения и т.д.)
2. Нажмите кнопку **«Экспорт»**
3. Выберите папку и имя файла
4. Откройте полученный `.xlsx`-файл в Microsoft Excel

### 🔍 Поиск
1. Введите текст в поле поиска (справа сверху)
2. Нажмите **«Поиск»** или `Enter`
3. Чтобы показать все записи — очистите поле и нажмите «Поиск» ещё раз

### 🕒 История изменений
1. В любом разделе нажмите кнопку **«История»**
2. Выберите запись в списке слева
3. Справа отобразятся:
   - Значения **до** изменения
   - Значения **после** изменения
   - Тип операции (Создание / Изменение / Удаление)
   - ФИО пользователя, внёсшего правку

## ❓ Частые вопросы

### ❓ Программа не запускается — «Не удалось найти .NET Framework»
✅ Установите .NET Framework 4.8: [Скачать](https://dotnet.microsoft.com/ru-ru/download/dotnet-framework/net48)

### ❓ Ошибка подключения к базе данных
✅ Проверьте:
- Запущен ли сервис SQL Server (в «Службах» Windows)
- Правильно ли указано имя сервера в `App.config` (по умолчанию: `.\SQLEXPRESS`)
- Существует ли база данных `PersonnelRecords`

### ❓ Не сохраняется запись в таблице
✅ Возможные причины:
- Не заполнено обязательное поле (например, `VacancyId` в резюме)
- Нарушено ограничение уникальности (дубликат `Login` или `Division+Post`)
- Нет прав на запись в БД (запустите приложение от имени администратора)

### ❓ Даты в Excel отображаются как `##########`
✅ Это не ошибка — просто колонка слишком узкая.  
Откройте файл в Excel → дважды кликните по границе заголовка колонки → ширина подстроится автоматически.

### ❓ Как восстановить удалённые данные?
✅ Откройте окно **«История изменений»**, найдите запись с типом «УДАЛЕНИЕ».  
Точные данные можно скопировать оттуда и создать запись заново.

## 🛠️ Для разработчиков и администраторов

### Структура проекта
```
PersonellRecords/
├── PersonellRecords.sln          # Решение Visual Studio
├── PersonellRecords/             # Основной проект
│   ├── Views/                    # Окна приложения (XAML + CS)
│   ├── Modules/                  # Бизнес-логика (ModuleDB, ModuleHistory...)
│   ├── App.xaml                  # Точка входа
│   └── App.config                # Строка подключения к БД
├── DatabaseScript.sql            # Скрипт создания БД
├── README.md                     # Этот файл
└── bin/Release/                  # Готовый исполняемый файл
```

### Строка подключения (App.config)
```xml
<connectionStrings>
  <add name="DefaultConnection" 
       connectionString="Data Source=.\SQLEXPRESS;Initial Catalog=PersonnelRecords;Integrated Security=True" 
       providerName="System.Data.SqlClient"/>
</connectionStrings>
```
При необходимости измените `Data Source` на имя вашего сервера.

### Сборка проекта
1. Откройте `PersonellRecords.sln` в Visual Studio 2022
2. Установите пакет через NuGet: `ClosedXML`
3. Сборка → **Построить решение** (`Ctrl+Shift+B`)
4. Готовый файл: `bin\Release\PersonellRecords.exe`
